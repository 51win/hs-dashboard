# 대시보드 v5 (백로그): 역할 분리 + 구글 시트 = DB

작성일: 2026-06-22
상태: 제안됨(다음 세션에서 구현) — 아직 미구현
관련: v4 스펙(`2026-06-22-dashboard-v4-sheets-statusline.spec.md`)을 확장/대체

## 목표 (사용자 요청)

- **구글 시트 = DB 적재용**(데이터 원본/저장소).
- **대시보드 뷰어 모드(관리자 아님)** = 읽기 전용. 시트 데이터를 보기만.
- **대시보드 관리자 모드** = 대시보드 안에서 직접 수정(편집) 가능 → 시트에 반영(쓰기).

즉 v4의 "읽기 전용만"에서 한 단계 확장: 같은 대시보드가 모드에 따라 뷰어/관리자로 동작.

## 구성 방향

- 모드 판별: 관리자 = 로그인(편집 권한)된 사용자, 뷰어 = 비로그인/공개.
  - 뷰어: 공개 시트를 gviz CSV로 읽어 표시(현재 v4 읽기 경로 재사용), 편집 UI 숨김.
  - 관리자: 시트 편집 권한으로 로그인 → 대시보드에서 수정 → 시트에 쓰기.
- 쓰기(관리자) 구현 옵션(다음 세션에 결정):
  1. **Google OAuth + Sheets API**(권장): GitHub Pages(https)에서 Google Identity로
     편집자 로그인 → Sheets API `values.update/append`로 행 갱신. (Google Cloud OAuth
     클라이언트 ID 필요, 승인 출처 = Pages URL.)
  2. Apps Script 웹앱 엔드포인트(시트에 붙인 doPost)로 쓰기 위임 — 클라이언트 비밀 없이
     가능하나 별도 배포 필요.
- 데이터 모델/시트 스키마는 v4 그대로(Tasks, Checklist 탭). 쓰기 시 같은 스키마로 역매핑.
- 충돌/동시편집: 우선 "로드 후 저장(낙관적)" + 새로고침 기반. 실시간 공동편집은 시트 자체에서.

## 모드 전환 UX(초안)

- 기본 = 뷰어(읽기 전용). 우상단 "관리자 로그인" → OAuth → 편집 모드 활성(에디터/추가/저장 노출).
- 관리자에서 "저장" = 시트에 쓰기(행 업데이트/추가) + 성공/실패 피드백.
- 로그아웃 시 뷰어로 복귀.

## 선행 조건(사용자 준비)

- Google Cloud 프로젝트 + OAuth 클라이언트 ID(웹), 승인된 JS 출처 = GitHub Pages URL.
- 시트 공유: 편집자(관리자) 소수 + 뷰어 공개 읽기.
- GitHub Pages 배포(https). (file://에서는 OAuth/API 불가.)

## 구현 시 작업 순서(예정)

1. v4 읽기 전용 경로를 "뷰어 모드"로 확정(이미 대부분 구현됨).
2. 관리자 인증(Google OAuth) 붙이기 → 편집 UI 게이팅.
3. 대시보드 편집 → 시트 쓰기(Sheets API 또는 Apps Script) 매핑/검증.
4. 모드 토글·세션 처리·에러 복구.
5. 헤드리스 테스트: 쓰기 매핑(데이터→행) 순수 함수 단위 테스트.
