# 편집 사이트 / 공유 사이트 분리 설계

날짜: 2026-06-23

## 배경

기존 `dashboard.html` 한 파일이 (1) 로컬 localStorage 편집, (2) 공개 구글 시트 읽기,
(3) Apps Script 비밀번호 쓰기(관리자 모드)를 모두 담고 있었다. "관리자/뷰어" 모드를
한 파일 안에서 분기(`canEdit()`)하다 보니 복잡해졌다.

## 목표

- 단순 열람자가 데이터를 수정할 수 없게 한다.
- 편집용 사이트(개인)와 공유용 사이트(공개)를 물리적으로 분리한다.
- 두 사이트는 **구글 시트 한 곳**을 통해 항상 동기화된다.

## 사용자 선택

- 데이터 저장소: **구글 시트 유지** (이미 읽기·쓰기 연동됨)
- 편집 사이트 위치: **내 컴퓨터 로컬 파일만**
- 공유 사이트 새로고침: **수동 새로고침 버튼**

## 아키텍처

데이터는 구글 시트에만 존재한다. 두 사이트가 같은 시트를 바라보므로 늘 동기화된다.

세 개의 정적 파일 + 토큰 사이드카:

- `dashboard-core.js` — 렌더링·데이터·시트 입출력 등 모든 로직. `window.Dashboard`를 노출.
  동작은 `window.DASHBOARD_MODE`(`"view"` | `"edit"`)로 결정. (기본값 `"edit"` — 테스트 호환)
  → 두 사이트가 **같은 코드**를 공유하므로 코드 차원에서도 어긋날 일이 없다.
- `index.html` — 공유(열람) 사이트. `DASHBOARD_MODE="view"`. 시트를 gviz CSV로 읽어 표시만 함.
  **편집기·저장·관리자 로그인·연결/내보내기 UI 없음.** "새로고침" 버튼으로 최신 시트 반영.
  GitHub Pages 루트로 배포해 팀에 공유.
- `edit.html` — 편집(개인) 사이트. `DASHBOARD_MODE="edit"`. 내 컴퓨터 로컬 전용.
  인라인 편집기 항상 활성. 변경은 localStorage에 즉시 저장(오프라인 안전) + 디바운스로
  시트에 자동 게시. `.gitignore`에 넣어 공개 URL이 생기지 않게 함.
- `dashboard.html` — `index.html`로 보내는 리다이렉트(기존 북마크/Pages 링크 호환).
- `dashboard-tokens.js` — 기존 토큰 사이드카(그대로).

## 데이터 동기화

- **올리기(편집 → 시트):** 편집 사이트에서 변경 발생 시 2.5초 디바운스로 Apps Script
  웹앱에 JSONP(GET, `<script>` 주입)로 전체 데이터를 전송 → 시트 덮어쓰기. JSONP는
  `file://`에서도 동작(외부 스크립트 로드는 CORS 비대상)하므로 로컬 편집 파일에서 게시 가능.
  수동 "지금 게시" 버튼과 상태 표시("저장 중…/시트에 반영됨 ✓ HH:MM/오류")도 둠.
  Apps Script는 비밀번호가 맞을 때만 기록(서버 검증). 비밀번호는 편집 사이트의
  localStorage에 보관(로컬 전용이므로 허용), 없으면 1회 입력. `unauthorized`면 지우고 재입력.
- **내려받기(시트 → 편집, 보조):** "시트에서 불러오기" 버튼은 gviz로 읽기를 시도.
  http(localhost/Pages)에서 열었을 때 동작. `file://`에서는 브라우저 제약으로 불가하며
  안내 메시지 표시. 단일 편집자 시나리오에선 평소 불필요.
- **공유 사이트(시트 → 표시):** https에서 gviz CSV로 읽음. "새로고침" 버튼으로 갱신.

## 모드별 동작 요약

| | view (index.html) | edit (edit.html) |
|---|---|---|
| 편집 UI | 없음 | 항상 표시 |
| 데이터 출처 | 시트(gviz) | localStorage(작업본) |
| 시트 쓰기 | 안 함 | 디바운스 자동 + 수동 게시 |
| 헤더 액션 | 새로고침 | 지금 게시 / 불러오기 / 내보내기·가져오기 |
| 관리자 모드 | 제거 | 제거(비밀번호는 게시용으로만) |

## 변경되는 부분

- `canEdit()`는 `DASHBOARD_MODE === "edit"`로 단순화. 관리자(login/logout) 개념 제거.
- 읽기전용/편집 혼합 분기 제거 — 모드가 정적으로 고정.
- `saveToSheet()`: 성공 후 gviz 재읽기(`loadFromSheet`) 호출 제거(`file://` 오류 방지),
  localStorage 작업본을 진실로 유지. 디바운스 래퍼 추가.
- 비밀번호: prompt(메모리) → localStorage 보관으로 변경.

## 테스트

- `tests/helpers.js`: `dashboard.html` 대신 `dashboard-core.js`를 주입한 JSDOM 문서를 만들고
  `loadDashboard(mode = "edit")`로 모드 지정 가능하게.
- 기존 편집 동작 테스트는 `mode="edit"`로 그대로 통과.
- 관리자 로그인 테스트(v5)는 새 모델로 교체: view 모드는 편집 UI 없음 / edit 모드는 편집 UI + 게시 버튼.
- 추가: view 모드 읽기 전용 렌더, edit 모드 자동 게시 디바운스 스케줄링(시간 mock) 검증.

## 비목표(YAGNI)

- 실시간 푸시/웹소켓, 다중 동시 편집 충돌 해결, OAuth 로그인은 범위 밖.
- 행 단위 증분 저장은 후속(현재는 전체 덮어쓰기 유지).
