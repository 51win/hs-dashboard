# 관리자 편집(쓰기) 설정 — Google OAuth + Sheets API

목표: GitHub Pages(https)에 올린 대시보드에서, **시트 편집 권한이 있는 구글 계정으로
로그인**한 관리자만 대시보드 안에서 수정 → 구글 시트에 저장.

관리자 판별 원칙(권장): **관리자 = 시트 편집 권한자.**
- 시트를 관리자 계정에만 "편집자"로 공유. 나머지는 공개 "뷰어"(읽기 전용).
- 대시보드는 로그인 사용자의 권한으로 Sheets API를 호출 → 편집 권한이 없으면 저장이 거부됨.
- 즉 구글 공유 설정이 곧 권한. 클라이언트 코드의 이메일 목록은 보안이 아니라 UI 편의일 뿐.

---

## 한승이 준비할 것 (Google Cloud Console)

1. https://console.cloud.google.com 에서 프로젝트 생성(또는 기존 선택).
2. "API 및 서비스 → 라이브러리"에서 **Google Sheets API** 사용 설정(Enable).
3. "API 및 서비스 → OAuth 동의 화면":
   - User Type = External, 앱 이름/이메일 입력.
   - 게시 상태가 "테스트"면 "테스트 사용자"에 **편집할 관리자 구글 이메일**을 추가
     (테스트 모드에선 등록된 사용자만 로그인 가능). 공개 게시는 선택.
   - 스코프는 별도 추가 안 해도 됨(코드에서 요청).
4. "API 및 서비스 → 사용자 인증 정보 → 사용자 인증 정보 만들기 → OAuth 클라이언트 ID":
   - 애플리케이션 유형 = **웹 애플리케이션**.
   - **승인된 JavaScript 원본**에 다음을 추가:
     - `https://51win.github.io`
     - (로컬 테스트용으로 쓸 거면) `http://localhost:8000` 등 실제 사용할 origin
   - 리디렉션 URI는 토큰 방식에선 필요 없음(비워도 됨).
   - 만들면 **클라이언트 ID**가 나옵니다(예: `xxxx.apps.googleusercontent.com`).
     클라이언트 보안 비밀은 사용하지 않습니다.
5. 시트 공유: 시트 "공유"에서 관리자 계정을 **편집자**로 추가(뷰어 공개는 유지).

준비되면 **클라이언트 ID** 문자열만 알려주세요. 그걸 대시보드 설정에 넣어 5·6단계를 구현합니다.

---

## 내가 구현할 것 (클라이언트 ID 받은 뒤)

- Google Identity Services(토큰 모델)로 "관리자 로그인" 버튼 → 스코프
  `https://www.googleapis.com/auth/spreadsheets` 액세스 토큰 획득.
- 로그인 시 편집 UI(에디터·추가·저장·작은 과제 추가/삭제) 노출, 로그아웃/뷰어 시 숨김.
- "저장" = 현재 데이터를 시트 행으로 역매핑해 Sheets API `values.update`로 기록
  (처음엔 전체 덮어쓰기, 이후 행 단위 최적화). 데이터→행 매핑은 헤드리스 테스트.
- 권한 없는 계정이 저장 시 API가 거부 → 안내 메시지.
- 클라이언트 ID는 코드 상수 또는 작은 설정 파일로 주입(공개돼도 되는 값이며, 보안은 origin
  제한 + 시트 권한으로 보장).

## 보안 메모

- 클라이언트 ID와 공개 시트 URL은 노출돼도 무방합니다(설계상 공개 읽기).
- 쓰기 권한은 "승인된 JS 원본(=Pages URL)" + "구글 계정의 시트 편집 권한" 두 겹으로 강제됩니다.
- 클라이언트 비밀 키는 정적 사이트에 두지 않습니다(토큰 방식이라 불필요).
